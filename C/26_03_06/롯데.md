# Spring Boot (Thymeleaf) + Vue.js 동적 메타 데이터 주입 가이드

본 문서는 Spring Boot 백엔드에서 조회한 메타 데이터를 Thymeleaf 템플릿 엔진을 매개체로 하여 Vue.js SPA(Single Page Application) 클라이언트에 동적으로 전달하는 아키텍처와 구현 방법을 설명합니다.

---

## 🗺️ 전체 아키텍처 흐름

```text
[사용자 요청] ➔ [Spring Boot 컨트롤러] ➔ [DB 데이터 조회]
                                              ↓
[브라우저 화면 렌더링] 🔀 [Thymeleaf 변수 치환] 🔀 [index.html에 데이터 주입]
        ↓ (Vue 구동)
[dataset에서 메타 정보 추출] ➔ [Vue 상태 관리 및 서비스 적용]
```

1. **사용자 요청:** 사용자가 특정 URL(예: `/products/10`)로 접근합니다.
2. **백엔드 처리:** Spring Boot 컨트롤러가 요청을 받아 비즈니스 데이터를 조회합니다.
3. **템플릿 렌더링:** 조회한 데이터를 Thymeleaf 변수에 담아 `index.html` 내의 `data-*` 속성에 주입합니다.
4. **프론트엔드 수신:** 브라우저에서 Vue 앱이 마운트되는 시점에 DOM의 `dataset`에서 데이터를 추출하여 전역 상태로 활용합니다.

---

## 🛠️ 단계별 구현 방법

### 1단계: Vue 프로젝트 설정 (`index.html`)

Vue 빌드의 뼈대가 되는 `index.html` 파일의 마운트 대상 태그(`<div id="app">`)에 타임리프 문법을 적용하여 서버 데이터를 받아올 그릇을 만듭니다.

* **파일 경로:** `public/index.html` (Vite 또는 Vue CLI 환경)

```html
<!DOCTYPE html>
<html lang="ko" xmlns:th="http://thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Vue & Spring Boot App</title>
</head>
<body>
    <!-- Spring Boot가 Thymeleaf 문법을 해석해 아래 속성에 값을 채워줍니다 -->
    <div
      id="app"
      th:data-travel-type="\${travelType}"
      th:data-product-type="\${productType}"
      th:data-ssr-info="\${metaData}"
    ></div>

    <script type="module" src="/src/main.js"></script>
</body>
</html>
```

> 💡 **빌드 파이프라인 참고:** 
> Vue 프로젝트를 빌드(`npm run build`)할 때 생성되는 최종 `index.html` 파일이 Spring Boot의 템플릿 경로(`src/main/resources/templates/`)로 자동 복사되도록 빌드 도구 설정을 연동해 두어야 합니다.

---

### 2단계: Spring Boot 백엔드 개발 (데이터 주입)

컨트롤러는 뷰 요청을 매핑하여 필요한 데이터를 조회한 뒤, 안전하게 JSON 문자열 등으로 변환하여 Model 객체에 담아 템플릿으로 전달합니다.

* **파일 경로:** `src/main/java/.../ViewController.java`

```java
import com.fasterxml.jackson.databind.ObjectMapper;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;

import java.util.HashMap;
import java.util.Map;

@Controller
public class WebViewController {

    private final ObjectMapper objectMapper = new ObjectMapper();

    @GetMapping("/products/{id}")
    public String renderVuePage(@PathVariable("id") Long id, Model model) {
        try {
            // 1. 비즈니스 로직 및 데이터 조회 (예시 데이터)
            String travelType = "DOMESTIC"; 
            String productType = "HOTEL";
            
            Map<String, Object> metaMap = new HashMap<>();
            metaMap.put("title", "제주도 3박 4일 힐링 패키지");
            metaMap.put("desc", "지금 바로 떠나는 특가 여행");
            metaMap.put("image", "https://example.com");

            // 2. 복잡한 Object는 ObjectMapper를 통해 안전한 JSON 문자열로 변환
            String metaDataJson = objectMapper.writeValueAsString(metaMap);

            // 3. Thymeleaf 변수명과 매핑하여 Model에 추가
            model.addAttribute("travelType", travelType);
            model.addAttribute("productType", productType);
            model.addAttribute("metaData", metaDataJson);

        } catch (Exception e) {
            // 예외 발생 시 기본값 세팅 또는 에러 페이지 처리
            model.addAttribute("travelType", "NONE");
            model.addAttribute("productType", "NONE");
            model.addAttribute("metaData", "{}");
        }

        // 4. src/main/resources/templates/index.html 을 렌더링
        return "index"; 
    }
}
```

### 3단계: Vue.js 프론트엔드 개발 (데이터 수신)

브라우저가 HTML을 로드하고 Vue 앱이 실행되는 구동 시점(Entry Point)에서 HTML DOM에 심겨 있는 `data-*` 속성을 추출합니다.

* **파일 경로:** `src/main.js` (또는 `src/App.vue`)

```javascript
import { createApp } from 'vue';
import App from './App.vue';

// 1. 타임리프에 의해 데이터가 주입된 DOM 엘리먼트 타겟팅
const appElement = document.getElementById('app');

if (appElement) {
    // 2. dataset 속성 추출 (html의 케밥케이스 속성은 자바스크립트에서 카멜케이스로 자동 변환됩니다)
    const travelType = appElement.dataset.travelType;
    const productType = appElement.dataset.productType;
    
    // 3. JSON 문자열 구조 복원 및 예외 처리
    let ssrInfo = {};
    try {
        ssrInfo = JSON.parse(appElement.dataset.ssrInfo || '{}');
    } catch (e) {
        console.error("서버 메타 데이터 파싱 실패:", e);
    }

    // 데이터 콘솔 확인 (개발 단계)
    console.log('서버 주입 데이터 수신 완료:', { travelType, productType, ssrInfo });

    // 4. Vue 인스턴스 생성 및 전역 프로퍼티(Global Properties)에 주입
    const app = createApp(App);
    
    app.config.globalProperties.\$serverData = {
        travelType,
        productType,
        ssrInfo
    };

    // 예시: 필요 시 Pinia 스토어가 있다면 초기값으로 주입할 수도 있습니다.
    // const store = useMainStore();
    // store.setServerData({ travelType, productType, ssrInfo });

    app.mount('#app');
}
```

## 💡 장점 및 주의사항 (Checklist)

### 장점
* **낮은 인프라 공수:** 완전한 SSR(Nuxt.js 등) 환경을 인프라에 별도로 구축하는 것에 비해 리소스와 공수가 현저히 적습니다.
* **통신 최적화:** API 서버에 데이터를 한 번 더 요청하는 2중 통신을 제거하므로 초기 페이지 진입 및 UI 렌더링 속도가 향상됩니다.

### 주의사항
* **XSS 공격 방어:** 사용자 입력 값이 포함된 데이터를 JSON 문자열로 넘길 때 HTML 태그 이스케이프 처리가 잘 되었는지 체크해야 합니다. (Thymeleaf 내부 직렬화나 Spring의 `HtmlUtils` 활용)
* **SEO(검색엔진 최적화) 한계:** 본 방식은 JavaScript가 읽을 수 있는 DOM의 '데이터 속성'에 값을 채우는 방식입니다. 따라서 구글봇은 파싱할 수 있으나, 카카오톡/페이스북 공유하기 봇처럼 **JS를 실행하지 않고 HTML `<head>` 안의 `<meta>` 태그만 긁어가는 크롤러**에게는 메타 정보가 노출되지 않을 수 있습니다. 
  * *만약 카카오톡 공유하기용 미리보기(OG Tag)가 주 목적인 경우, 본 방식과 병행하여 HTML `<head>` 안에도 `th:content="${...}"` 형태의 메타 태그를 직접 배치해야 합니다.*
